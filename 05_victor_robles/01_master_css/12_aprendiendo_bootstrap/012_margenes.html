Margin and padding

Asigna valores o adaptables margin o padding a un elemento o a un subconjunto de sus lados con clases abreviadas. Incluye compatibilidad con propiedades individuales, todas las propiedades y propiedades verticales y horizontales. Las clases se construyen a partir de un mapa Sass predeterminado que va desde .25rem hasta 3rem .

¿Usas el módulo de diseño CSS Grid? Considera usar la utilidad gap.

Notación

Las utilidades de espaciado que se aplican a todos los puntos de interrupción, desde xshasta xxl, no incluyen abreviatura. Esto se debe a que estas clases se aplican desde min-width: 0y hacia arriba, por lo que no están vinculadas por una consulta de medios. Sin embargo, los puntos de interrupción restantes sí incluyen una abreviatura.

Las clases se nombran utilizando el formato {property}{sides}-{size} para xs y {property}{sides}-{breakpoint}-{size}para sm, md, lg, xl, y xxl.

Cuando la propiedad es una de las siguientes:

• m- para clases que establecen margin.

• p- para clases que establecen padding.

Donde lados es uno de:

• t- (top)para clases que establecen margin-top o padding-top.

• b- (bottom)para clases que establecen margin-bottom o padding-bottom.

• s- (inicio)para clases que establecen margin-left o padding-left LTR margin-right o padding-right en RTL.

• e- (inicio)para clases que establecen margin-right o padding-right LTR margin-left o padding-left en RTL.

• x- para clases que establecen margin.

• y- para clases que establecen margin.

• en blanco- para clases que establecen margin o padding en los cuatro elementos.

Donde el tamaño es uno de los siguientes:

• 0- para clases que eliminan el margin o padding configurándolo en 0 .

• 1- para clases que establecen el margin o padding configurándolo en $spacer * .25.

• 2- para clases que establecen el margin o padding configurándolo en $spacer * .5.

• 3- para clases que establecen el margin o padding configurándolo en $spacer * 1.

• 4- para clases que establecen el margin o padding configurándolo en $spacer * 1.5.

• 5- para clases que establecen el margin o padding configurándolo en $spacer * 3.

• auto- para clases que configuran el margin o padding en automático.

(Puede agregar más tamaños agregando entradas a la variable $spacer en el mapa Sass)

Ejemplos

A continuación se muestran unos ejemplos representativos de estas clases:

.mt-0 {
        margin-top: 0 !important;
        }

.ms-1 {
        margin-left: ($spacer * .25) !important;
        }

.px-2 {
        padding-left: ($spacer * .5) !important;
        padding-right: ($spacer * .5) !important;
        }

.p-3 {
        padding: $spacer !important;
        }

Negative margin/Margen Negativo

En CSS, las propiedades de margin pueden usar valores negativos ( padding no se pueden). Estos márgenes negativos están deshabilitados por defecto , pero se pueden habilitar en Sass configurando [Nombre del elemento] $enable-negative-margins: true.
La sintaxis es prácticamente la misma que la de las utilidades de margen positivo predeterminadas, pero con la adición de n antes del tamaño solicitado. Aquí hay una clase de ejemplo que es lo opuesto a .mt-1:

                
Margen Negativo.

            
        
            .mt-n1 {
                margin-top: -0.25rem !important;
            }

Esto le ahorra tener que agregar utilidades de margen a elementos secundarios individuales de una cuadrícula o contenedor flexible. Las utilidades de Gap son adaptables por defecto y se generan a través de nuestra API de utilidades, basada en el $spacers mapa de Sass.

Sass maps

Las utilidades de espaciado se declaran a través del mapa Sass y luego se generan con nuestra API de utilidades.

scss/variables.scss 
 
        $spacer: 1rem; 
        $spacers: (
            0: 0, 
            1: $spacer * .25, 
            2: $spacer * .5, 
            3: $spacer, 
            4: $spacer * 1.5, 
            5: $spacer * 3, 
        );
