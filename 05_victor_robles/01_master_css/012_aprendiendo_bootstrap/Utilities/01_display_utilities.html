Las utilidades de Bootstrap se generan con nuestra API de utilidades y pueden usarse para modificar o ampliar nuestro conjunto predeterminado de clases de utilidad mediante Sass. Nuestra API de utilidades se basa en una serie de mapas y funciones de Sass para generar familias de clases con diversas opciones. Si no está familiarizado con los mapas de Sass, consulte la documentación oficial de Sass para comenzar.

El mapa de $utilities contiene todas nuestras utilidades y posteriormente se fusiona con su mapa de $utilities personalizado, si lo tiene. El mapa de utilidades contiene una lista de grupos de utilidades con claves que aceptan las siguientes opciones:

MAPA DE UTILITIES
Opción Tipo Valor Predeterminado Descripción
property. Requerido -- Nombre de la propiedad, que puede ser una cadena o una matriz de cadenas (por ejemplo, rellenos o márgenes horizontales).
values Requerido 30 30
class Opcional 30 40
css-var Opcional nulo Nombre de la clase generada. Si no se proporciona y property es un array de cadenas, class se usará el primer elemento del property array por defecto. Si no se proporciona y property es una cadena, valuesse usan las claves para los classnombres.
css-var Opcional false Booleano para generar variables CSS en lugar de reglas CSS.
css-variable-name Opcional nulo Nombre personalizado sin prefijo para la variable CSS dentro del conjunto de reglas.
local-vars Opcional nulo Mapa de variables CSS locales para generar además de las reglas CSS.
state Opcional nulo Lista de variantes de pseudoclase (por ejemplo, :hover o :focus ) para generar.
responsive Opcional false Valor booleano que indica si se deben generar clases responsivas.
rfs Opcional flse Booleano para habilitar el reescalado de fluidos con RFS .
print Opcional false Valor booleano que indica si es necesario generar clases de impresión.
rtl Opcional true Valor booleano que indica si la utilidad debe mantenerse en RTL.

API explained/ Explicación de la API

Todas las variables de utilidad se añaden a la $utilities variable dentro de nuestra _utilities.scss hoja de estilos. Cada grupo de utilidades tiene un aspecto similar al siguiente:

$utilities : (
    "opacity": (
    property: opacity,
    values: (
      0: 0,
      25: .25,
      50: .5,
      75: .75,
      100: 1,
    )
  )
);

Lo cual produce el siguiente resultado: 

.opacity-0 { opacity: 0; }
.opacity-25 { opacity: .25; }
.opacity-50 { opacity: .5; }
.opacity-75 { opacity: .75; }
.opacity-100 { opacity: 1; }

Property / Propiedad

La clave requerida property debe estar establecida para cualquier utilidad y debe contener una propiedad CSS válida. Esta propiedad se utiliza en el conjunto de reglas de la utilidad generada. Siclass se omite la clave, también sirve como nombre de clase predeterminado. Considere la utilidad text-decoration :

$utilities:(
"text-decoration": (
    property:text-decoration,
    values:none underline line-through 
)
      );

.text-decoration-none { text-decoration: none !important; }
.text-decoration-underline { text-decoration: underline !important; }
.text-decoration-line-through { text-decoration: line-through !important; }

Values/ Valores

Utilice lavalues lave para especificar qué valores de la clase especificadaproperty deben usarse en los nombres y reglas de clase generados. Puede ser una lista o un mapa (configurado en las utilidades o en una variable de Sass).

Como lista, al igual que con las utilidades text-decoration : 

 values: none underline line-through

Como mapa, al igual que con las utilidades opacity:

values: (
    0: 0,
    25: .25,
    50: .5,
    75: .75,
    100: 1,
)

Como una variable Sass que establece la lista o el mapa, como en nuestras utilidades de posición:

values: $position-values

Class/ Clase

Utilice la opción class para cambiar el prefijo de clase usado en el CSS compilado. Por ejemplo, para cambiar de .opacity-* a .o-* : 

$utilities:(
    "opacity": (
        property: opacity,
        class: o ,
        values: (
        0: 0,
        25: .25,
        50: .5,
        75: .75,
        100: 1,
        )
    )
);

Producción:

    .o-0 { opacity:  0!important; }
    .o-25 { opacity:  .25!important; }
    .o-50 { opacity:  .5!important; }
    .o-75 { opacity:  .75!important; }
    .o-100 { opacity:  1!important; }

Si class: null, genera clases para cada una de las claves values:

$utilities:(
    "visibility": (
        property:visibility,
        class: null,
        values: (
        visible: visible,
        invisible: hidden,
        )
    )
);

Producción: 

.visible { visibility: visible !important; }
.invisible { visibility: visible !important; }

CSS variable utilities / Utilidades de variables CSS

Establezca la opción booleana css-var en true y la API generará variables CSS locales para el selector dado, en lugar de las reglas habituales property: value . Agregue una opción css-variable-name para establecer un nombre de variable CSS diferente al nombre de la clase.

Considere nuestras utilidades .text-opacity-* . Si añadimos la opción css-variable-name , obtendremos una salida personalizada. 

$utilities:(
    "text-opacity": (
        css-var:true,
        css-variable-name: text-alpha,
        class: text-opacity,
        values: (
            25: .25,
            50: .5,
            75: .75,
            100: 1
        )
    ),
);

Producción: 

    .text-opacity-25 { --bs-text-alpha:.25;}
    .text-opacity-50 { --bs-text-alpha: .5;}
    .text-opacity-75 { --bs-text-alpha: .75;}
    .text-opacity-100 { --bs-text-alpha: 1;}

Local CSS variables / Variables CSS locales

Utilice la opción local-vars para especificar un mapa Sass que genere variables CSS locales dentro del conjunto de reglas de la clase de utilidad. Tenga en cuenta que consumir dichas variables CSS locales en las reglas CSS generadas puede requerir trabajo adicional. Por ejemplo, considere nuestras utilidades .bg-* :

$utilities: (
    "background-color": (
    property: background-color,
    class: bg,
    local-vars: (
        "bg-opacity": 1
    ),
    values: map-merge(
        $utilities-bg-colors,
        (
        "transparent": transparent
        )
    )
)
);

Producción: 

        .bg-primary {
            --bs-bg-opacity: 1;
            background-color: rgba(var(--bs-primary-rgb), var(--bs-bg-opacity)) !important;
          }

States/ Estados

La clave requerida Utilice la opción state para generar variaciones de pseudoclases. Ejemplos de pseudoclases son state :hovery :state focus. Al proporcionar una lista de estados, se crean nombres de clase para esa pseudoclase. Por ejemplo, para cambiar la opacidad al pasar el cursor, añada state: hover y obtendrá el resultado .opacity-hover:hover en su CSS compilado.

¿Necesita varias pseudoclases? Use una lista de estados separados por espacios: state: hover focus. 

$utilities: (
    "opacity": (
      property: opacity,
      class: opacity,
      state: hover,
      values: (
        0: 0,
        25: .25,
        50: .5,
        75: .75,
        100: 1,
      )
    )
  );

Producción: 

.opacity-0-hover:hover { opacity: 0 !important; }
.opacity-25-hover:hover { opacity: .25 !important; }
.opacity-50-hover:hover { opacity: .5 !important; }
.opacity-75-hover:hover { opacity: .75 !important; }
.opacity-100-hover:hover { opacity: 1 !important; }

Responsive / Sensible

Agregue el valor booleano responsive para generar utilidades receptivas (por ejemplo, .opacity-md-25) en todos los puntos de interrupción.

$utilities: (
    "opacity": (
      property: opacity,
      responsive: true,
      values: (
        0: 0,
        25: .25,
        50: .5,
        75: .75,
        100: 1,
      )
    )
  );

Print/ Imprimir

Al habilitar la opción print también se generarán clases de utilidad para imprimir, que solo se aplican dentro de la consulta de medios @media print { ... } .

$utilities: (
    "opacity": (
      property: opacity,
      print: true,
      values: (
        0: 0,
        25: .25,
        50: .5,
        75: .75,
        100: 1,
      )
    )
  );

Producción: 

.opacity-0 { opacity: 0 !important; }
.opacity-25 { opacity: .25 !important; }
.opacity-50 { opacity: .5 !important; }
.opacity-75 { opacity: .75 !important; }
.opacity-100 { opacity: 1 !important; }

.opacity-print-0 { opacity: 0 !important; }
.opacity-print-25 { opacity: .25 !important; }
.opacity-print-50 { opacity: .5 !important; }
.opacity-print-75 { opacity: .75 !important; }
.opacity-print-100 { opacity: 1 !important; }

Important / Importancia

Todas las utilidades generadas por la API se incluyen !important para garantizar que anulen los componentes y las clases modificadoras según lo previsto. Puede activar o desactivar esta configuración globalmente con la variable $enable-important-utilities (el valor predeterminado es true).

Using the API / Usando la API

Ahora que está familiarizado con el funcionamiento de la API de utilidades, aprenda a agregar sus propias clases personalizadas y modificar nuestras utilidades predeterminadas.

Override utilities / Anular utilidades

Anule las utilidades existentes usando la misma clave. Por ejemplo, si desea clases de utilidades de desbordamiento con capacidad de respuesta adicionales, puede hacer lo siguiente:

$utilities: (
    "overflow": (
      responsive: true,
      property: overflow,
      values: visible hidden scroll auto,
    ),
  );

Add Utilities/ Agregar utilidades

Se pueden agregar nuevas utilidades $utilities al mapa predeterminado con un map-merge. Asegúrese de utilities _.scssimportar primero los archivos Sass necesarios y luego use el map-merge map-mergepara agregar las utilidades adicionales. Por ejemplo, aquí se explica cómo agregar una utilidad cursor adaptable con tres valores.

@import "bootstrap/scss/functions";
@import  "bootstrap/scss/variables";
@import  "bootstrap/scss/variables-dark";
@import  "bootstrap/scss/maps";
@import  "bootstrap/scss/mixins";
@import  "bootstrap/scss/utilities";

$utilities: map-merge(
    $utilities,
  (
    "cursor": (
      property: cursor,
      class: cursor,
      responsive: true,
      values: auto pointer grab,
    )
  )
);

@import "bootstrap/scss/utilities/api";

Modify Utilities / Modificar utilidades

Modifique las utilidades existentes en el mapa $utilitiespredeterminado con las funciones map-get y map-merge. En el siguiente ejemplo, añadimos un valor adicional a las utilidades width. Comience con una inicial map-merge y luego especifique la utilidad que desea modificar. A partir de ahí, obtenga el "width"mapa anidado con map-get para acceder y modificar las opciones y valores de la utilidad.

@import "bootstrap/scss/functions";
@import  "bootstrap/scss/variables";
@import  "bootstrap/scss/variables-dark";
@import  "bootstrap/scss/maps";
@import  "bootstrap/scss/mixins";
@import  "bootstrap/scss/utilities";

  $utilities: map-merge(
    $utilities,
    (
    "width": map-merge(
      map-get($utilities, "width"),
        (
        values: map-merge(
        map-get(map-get($utilities, "width"), "values"),
                    (10: 10%),
                  ),
                ),
              ),
            )
          );
              
@import  "bootstrap/scss/utilities/api";

Enable responsive/ Habilitar la capacidad de respuesta

Puedes habilitar clases responsivas para un conjunto de utilidades que actualmente no responden de forma predeterminada. Por ejemplo, para que las clases border respondan:

@import "bootstrap/scss/functions";
@import  "bootstrap/scss/variables";
@import  "bootstrap/scss/variables-dark";
@import  "bootstrap/scss/maps";
@import  "bootstrap/scss/mixins";
@import  "bootstrap/scss/utilities";

$utilities: map-merge(
    $utilities,(
    "border": map-merge(
      map-get($utilities, "border"),
       ( responsive: true ), 
              ),
            )                           
          );                             
@import "bootstrap/scss/utilities/api";

Esto generará variaciones adaptables de .border y .border-0 para cada punto de interrupción.
El CSS generado se verá así:

    .border-0 { ... }
    
    @media (min-width: 576px) {
      
      .border-sm { ... }
      .border-sm-0 { ... }
    }
    
    @media (min-width: 768px) {
      
      .border-md { ... }
      .border-md-0 { ... }
    }
    
    @media (min-width: 992px) {
      .border-lg { ... }
      .border-lg-0 { ... }
      .border-lg { ... }
      .border-lg-0 { ... }
    }
    
    @media (min-width: 1200px) {
      .border-xl { ... }
      .border-xl-0 { ... }
      .border-xl { ... }
      .border-xl-0 { ... }


    }

        .border-xxl { ... }

    }

Rename utilities/ Cambiar el nombre de las utilidades

¿Faltan utilidades de la versión 4 o se han usado otras convenciones de nomenclatura? La API de utilidades permite anular el resultado de una utilidad classdeterminada; por ejemplo, para renombrar .ms-*las utilidades a oldish(algo viejo) .ml-*:

@import "bootstrap/scss/functions";
@import  "bootstrap/scss/variables";
@import  "bootstrap/scss/variables-dark";
@import  "bootstrap/scss/maps";
@import  "bootstrap/scss/mixins";
@import  "bootstrap/scss/utilities";

$utilities: map-merge(
    $utilities,(
        "margin-start": map-merge(
            map-get($utilities, "margin-start"),
       ( class: ml ), 
              ),
            )                           
          );                             
@import "bootstrap/scss/utilities/api";

Remove utilities/Eliminar utilidades

Elimina cualquiera de las utilidades predeterminadas con la map-remove()función Sass.

@import "bootstrap/scss/functions";
@import  "bootstrap/scss/variables";
@import  "bootstrap/scss/variables-dark";
@import  "bootstrap/scss/maps";
@import  "bootstrap/scss/mixins";
@import  "bootstrap/scss/utilities";

// Remove multiple utilities with a comma-separated list
    $utilities: map-remove($utilities, "width", "float");
                               
@import "bootstrap/scss/utilities/api";

También puedes utilizar la map-merge()función Sass y establecer la clave de grupo para eliminar la utilidad null.

@import "bootstrap/scss/functions";
@import  "bootstrap/scss/variables";
@import  "bootstrap/scss/variables-dark";
@import  "bootstrap/scss/maps";
@import  "bootstrap/scss/mixins";
@import  "bootstrap/scss/utilities";

$utilities: map-merge(
    $utilities,
    (
        "width":null 
    )                           
);                             
@import "bootstrap/scss/utilities/api";

Add, remove, modify / Agregar, eliminar, modificar

@import "bootstrap/scss/functions";
@import  "bootstrap/scss/variables";
@import  "bootstrap/scss/variables-dark";
@import  "bootstrap/scss/maps";
@import  "bootstrap/scss/mixins";
@import  "bootstrap/scss/utilities";

$utilities: map-merge(
    $utilities,
    (
    // Remove the `width` utility
    "width":null 

    // Make an existing utility responsive
    "border": map-merge(
        map-get($utilities, "border"),
      ( responsive: true ),
),

    // Add new utilities
    "cursor": (
      property: cursor,
      class: cursor,
      responsive: true,
      values: auto pointer grab,
    )
  )
);
                         
@import "bootstrap/scss/utilities/api";

Remove utility in RTL / Eliminar utilidad en RTL

Algunos casos extremos dificultan el estilo RTL , como los saltos de línea en árabe. Por lo tanto, se pueden omitir las utilidades de la salida RTL configurando la opción rtl en false :

            $utilities: (
                "word-wrap": (
                  property: word-wrap word-break,
                  class: text,
                  values: (break: break-word ),
                  rtl: false
                ),
              );

Producción: 

        /* rtl:begin:remove */
        .text-break  {
        word-wrap: break-word !important;   
        word-wrap: break-word !important;   
        }
        /* rtl:end:remove */

Esto no genera nada en RTL, gracias a la directiva de control RTLCSSremove .
