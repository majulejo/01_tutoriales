DISPLAY/MOSTRAR PROPIEDAD

Cambie rápidamente y con rapidez el valor de visualización de los componentes y mucho más con nuestras utilidades de visualización. Incluye compatibilidad con algunos de los valores más comunes, así como funciones adicionales para controlar la visualización al imprimir.

How it works?/ ¿Como funciona?

Cambie el valor de la propiedad display con nuestras clases de utilidad de visualización adaptables. Admitimos solo un subconjunto de todos los valores posibles para [nombre del elemento] display. Las clases se pueden combinar para lograr diversos efectos según sus necesidades.

Notation/ Notación

Colorizar el texto con utilidades en color. Si desea colorear enlaces, puede utilizar el .link-*clases de ayudantes que tienen :hover y estados :focus .

Las clases de utilidad de visualización que se aplican a todos los puntos de interrupción , desde xshasta xxl, no tienen abreviatura de punto de interrupción. Esto se debe a que dichas clases se aplican desde min-width: 0;y hacia arriba, y por lo tanto no están vinculadas por una consulta de medios. Sin embargo, los puntos de interrupción restantes sí incluyen una abreviatura de punto de interrupción.

Como tal, las clases se nombran utilizando el formato:

  • .d-{value}paraxs
  • .d-{breakpoint}-{value} para sm, md, lg, xl, y xxl.

Donde valor es uno de los siguientes:

  • none
  • inline
  • inline-block
  • block
  • grid
  • inline-grid
  • table
  • table-cell
  • table-row
  • flex
  • inline-flex

Los valores de visualización se pueden modificar modificando los valores display definidos $utilities y volviendo a compilar el SCSS.

Las consultas de medios afectan a anchos de pantalla con el punto de interrupción indicado o mayor . Por ejemplo, con .d-lg-none, se establece display: none , en las pantallas lg,xl y xxl

Specificity/ Especificidad

A veces, las clases contextuales no se pueden aplicar debido a la especificidad de otro selector. En algunos casos, una solución alternativa suficiente es encapsular el contenido del elemento en uno <div> o más elementos semánticos con la clase deseada.

Sass maps/Mapas de Sass

Luego, los colores del tema se colocan en un mapa Sass para que podamos recorrerlos para generar nuestras utilidades, modificadores de componentes y más.

scss/_variables.scss 
    $theme-colors: (
        "primary":    $primary,
        "secondary":  $secondary,
        "success":    $success,
        "info":       $info,
        "warning":    $warning,
        "danger":     $danger,
        "light":      $light,
        "dark":       $dark
    );

Los colores en escala de grises también están disponibles como mapa Sass. Este mapa no se utiliza para generar utilidades.

scss/_variables.scss 
    $grays: (
        "100": $gray-100,
        "200": $gray-200,
        "300": $gray-300,
        "400": $gray-400,
        "500": $gray-500,
        "600": $gray-600,
        "700": $gray-700,
        "800": $gray-800,
        "900": $gray-900
        );

Los colores RGB se generan a partir de un mapa Sass independiente:

scss/_maps.scss 
    $theme-colors-rgb: map-loop($theme-colors, to-rgb, "$value");

Las opacidades del color de fondo se basan en esto con su propio mapa que es consumido por la API de utilidades:

scss/_maps.scss 
     $utilities-bg: map-merge(

        (
            "black": to-rgb( $black),
            "white": to-rgb( $white),
            "body": to-rgb( $body-bg)
        )
    );
     $utilities-bg-colors: map-loop( $utilities-bg,  rgba-css-var, "$key", "bg");
      
     $utilities-bg-subtle: (
        "primary-subtle": var(--#{ $prefix}primary-bg-subtle),

Sass utilities API/API de utilidades de Sass

Las utilidades de color se declaran en nuestra API de utilidades en [nombre del archivo scss/_utilities.scss. Aprenda a usar la API de utilidades.

scss/_utilitiess.scss 
            "color": (
            property: color,
            class: text,
            local-vars: (
              "text-opacity": 1
            ),
            values: map-merge(
              $utilities-text-colors,
              (
    "muted":    var(--#{$prefix}secondary-color), // deprecated
    "black-50":    rgba($black, .5), // deprecated
    "white-50":   rgba($white, .5), // deprecated
    "body-secondary":     var(--#{$prefix}secondary-color),
    "body-tertiary": <   var(--#{$prefix}tertiary-color),
    "body-emphasis":  var(--#{$prefix}emphasis-color),
    "reset":    inherit,
              )
            )
          ),
          "text-opacity": (
            css-var: true,
            class: text-opacity,
            values: (
              25: .25,
              50: .5,
              75: .75,
              100: 1
            )
          ),
          "text-color": (
            property: color,
            class: text,
            values: $utilities-text-emphasis-colors
          ),
