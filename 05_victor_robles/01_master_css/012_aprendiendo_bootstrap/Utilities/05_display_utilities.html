Las utilidades de Bootstrap se generan con nuestra API de utilidades y pueden usarse para modificar o ampliar nuestro conjunto predeterminado de clases de utilidad mediante Sass. Nuestra API de utilidades se basa en una serie de mapas y funciones de Sass para generar familias de clases con diversas opciones. Si no está familiarizado con los mapas de Sass, consulte la documentación oficial de Sass para comenzar.

El mapa de $utilities contiene todas nuestras utilidades y posteriormente se fusiona con su mapa de $utilities personalizado, si lo tiene. El mapa de utilidades contiene una lista de grupos de utilidades con claves que aceptan las siguientes opciones:

MAPA DE UTILITIES
Opción Tipo Valor Predeterminado Descripción
property. Requerido -- Nombre de la propiedad, que puede ser una cadena o una matriz de cadenas (por ejemplo, rellenos o márgenes horizontales).
values Requerido 30 30
class Opcional 30 40
css-var Opcional nulo Nombre de la clase generada. Si no se proporciona y property es un array de cadenas, class se usará el primer elemento del property array por defecto. Si no se proporciona y property es una cadena, valuesse usan las claves para los classnombres.
css-var Opcional false Booleano para generar variables CSS en lugar de reglas CSS.
css-variable-name Opcional nulo Nombre personalizado sin prefijo para la variable CSS dentro del conjunto de reglas.
local-vars Opcional nulo Mapa de variables CSS locales para generar además de las reglas CSS.
state Opcional nulo Lista de variantes de pseudoclase (por ejemplo, :hover o :focus ) para generar.
responsive Opcional false Valor booleano que indica si se deben generar clases responsivas.
rfs Opcional flse Booleano para habilitar el reescalado de fluidos con RFS .
print Opcional false Valor booleano que indica si es necesario generar clases de impresión.
rtl Opcional true Valor booleano que indica si la utilidad debe mantenerse en RTL.

API explained/ Explicación de la API

Todas las variables de utilidad se añaden a la $utilities variable dentro de nuestra _utilities.scss hoja de estilos. Cada grupo de utilidades tiene un aspecto similar al siguiente:

$utilities : (
    "opacity": (
    property: opacity,
    values: (
      0: 0,
      25: .25,
      50: .5,
      75: .75,
      100: 1,
    )
  )
);

Lo cual produce el siguiente resultado: 

.opacity-0 { opacity: 0; }
.opacity-25 { opacity: .25; }
.opacity-50 { opacity: .5; }
.opacity-75 { opacity: .75; }
.opacity-100 { opacity: 1; }

Property / Propiedad

La clave requerida property debe estar establecida para cualquier utilidad y debe contener una propiedad CSS válida. Esta propiedad se utiliza en el conjunto de reglas de la utilidad generada. Siclass se omite la clave, también sirve como nombre de clase predeterminado. Considere la utilidad text-decoration :

$utilities:(
"text-decoration": (
    property:text-decoration,
    values:none underline line-through 
)
      );

.text-decoration-none { text-decoration: none !important; }
.text-decoration-underline { text-decoration: underline !important; }
.text-decoration-line-through { text-decoration: line-through !important; }

Values/ Valores

Utilice lavalues lave para especificar qué valores de la clase especificadaproperty deben usarse en los nombres y reglas de clase generados. Puede ser una lista o un mapa (configurado en las utilidades o en una variable de Sass).

Como lista, al igual que con las utilidades text-decoration : 

 values: none underline line-through

Como mapa, al igual que con las utilidades opacity:

values: (
    0: 0,
    25: .25,
    50: .5,
    75: .75,
    100: 1,
)

Como una variable Sass que establece la lista o el mapa, como en nuestras utilidades de posición:

values: $position-values

Class/ Clase

Utilice la opción class para cambiar el prefijo de clase usado en el CSS compilado. Por ejemplo, para cambiar de .opacity-* a .o-* : 

    <div class="p-3 mb-2  bg-transparent text-body">.bg-transparent</div>

Producción:

            .o-0 { opacity: 0 !important; }
            .o-25 { opacity: .25 !important; }
            .o-50 { opacity: .5 !important; }
            .o-75 { opacity: .75 !important; }
            .o-100 { opacity: 1 !important; }

Si class: null, genera clases para cada una de las valuesclaves: 

            $utilities: (
                "visibility": (
                  property: visibility,
                  class: null,
                  values: (
                    visible: visible,
                    invisible: hidden,
                  )
                )
              );

Producción: 

            .visible { visibility: visible !important; }
            .invisible { visibility: hidden !important; }

Opacity /Opacidad

A partir de la versión 5.1.0, las utilidades de background-color se generan con Sass mediante variables CSS. Esto permite cambios de color en tiempo real sin compilación y cambios dinámicos de transparencia alfa. .

¿Como funciona?

Consideremos nuestra utilidad predeterminada .bg-success.

Fondo verde con opacidad al 50%

Fondo verde con opacidad al 50%

En el <head> hemos definido el color --bs-success-rgb: 25, 135, 84; dentro del root.

    <style> 
        /* Definir la variable --bs-success-rgb */
        :root {
            --bs-success-rgb: 25, 135, 84; /* RGB para el color verde éxito (#198754) */
        }
    </style>
    
    .bg-success {
        --bs-bg-opacity: 1;
        background-color: rgba(var(--bs-success-rgb), var(--bs-bg-opacity)) !important;
    }

Usamos una versión RGB de nuestra variable CSS --bs-success (con el valor 25, 135, 84 ) y adjuntamos una segunda variable CSS, --bs-bg-opacity , para la transparencia alfa (con un valor predeterminado 1 gracias a una variable CSS local). Esto significa que, al usar .bg-success `now`, el color valor calculado es rgba(25, 135, 84, 1) . La variable CSS local dentro de cada .bg-* clase evita problemas de herencia, por lo que las instancias anidadas de las utilidades no modifican automáticamente la transparencia alfa.

Sin opacidad (.bg-success)
Con opacidad personalizada (.custom-opacity)
Sobre un fondo claro

HTML

    .bg-success {
        --bs-bg-opacity: 1;
        background-color: rgba(var(--bs-success-rgb), var(--bs-bg-opacity)) !important;
      }

Sass maps/Mapas de Sass

Luego, los colores del tema se colocan en un mapa Sass para que podamos recorrerlos para generar nuestras utilidades, modificadores de componentes y más.

scss/_variables.scss 
    $theme-colors: (
        "primary":    $primary,
        "secondary":  $secondary,
        "success":    $success,
        "info":       $info,
        "warning":    $warning,
        "danger":     $danger,
        "light":      $light,
        "dark":       $dark
    );

Los colores en escala de grises también están disponibles como mapa Sass. Este mapa no se utiliza para generar utilidades.

scss/_variables.scss 
    $grays: (
        "100": $gray-100,
        "200": $gray-200,
        "300": $gray-300,
        "400": $gray-400,
        "500": $gray-500,
        "600": $gray-600,
        "700": $gray-700,
        "800": $gray-800,
        "900": $gray-900

Los colores RGB se generan a partir de un mapa Sass independiente:

scss/_maps.scss 
    $theme-colors-rgb: map-loop($theme-colors, to-rgb, "$value");
