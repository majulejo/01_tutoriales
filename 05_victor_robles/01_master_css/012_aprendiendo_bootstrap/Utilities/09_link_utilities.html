Consejo de accesibilidad: Usar color para añadir significado solo proporciona una indicación visual, que no se transmitirá a los usuarios de tecnologías de asistencia como los lectores de pantalla. Asegúrese de que el significado sea evidente en el propio contenido (por ejemplo, texto visible con suficiente contraste de color ) o que se incluya mediante medios alternativos, como texto adicional oculto en la clase .visually-hidden.

CSS

Además de la siguiente funcionalidad de Sass, considere leer acerca de nuestras propiedades personalizadas CSS incluidas (también conocidas como variables CSS) para colores y más.

Sass utilities API/API de utilidades de Sass

Las utilidades de interacción se declaran en nuestra API de utilidades en [nombre del archivo] scss/_utilities.scss. Aprenda a usar la API de utilidades. incluidas (también conocidas como variables CSS) para colores y más.

scss/_utilities.scss 
"link-opacity": (
    css-var: true,
    class: link-opacity,
    state: hover,
    values: (
      10: .1,
      25: .25,
      50: .5,
      75: .75,
      100: 1
    )
  ),
  "link-offset": (
    property: text-underline-offset,
    class: link-offset,
    state: hover,
    values: (
      1: .125em,
      2: .25em,
      3: .375em,
    )
  ),
  "link-underline": (
    property: text-decoration-color,
    class: link-underline,
    local-vars: (
      "link-underline-opacity": 1
    ),
    values: map-merge(
      $utilities-links-underline,(
        null: rgba(var(--#{$prefix}link-color-rgb), var(--#{$prefix}link-underline-opacity, 1)),
      )
    )
  ),
  "link-underline-opacity": (
    css-var: true,
    class: link-underline-opacity,
    state: hover,
    values: (
      0: 0,
      10: .1,
      25: .25,
      50: .5,
      75: .75,
      100: 1
    ),
  ),
