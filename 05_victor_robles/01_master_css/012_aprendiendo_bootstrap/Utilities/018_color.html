COLORS

Transmita el significado a través de color con un puñado de clases de servicios públicos de color.
Incluye apoyo para los vínculos de estilo con los estados flotantes, también.

Consejo de accesibilidad: Usar el color para añadir significado sólo proporciona una indicación visual, que no será transmitida a los usuarios de tecnologías de asistencia como lectores de pantalla. Por favor, asegúrese de que el significado es obvio por el contenido en sí (por ejemplo, el texto visible con un suficiente contraste de color ) o que se incluya mediante medios alternativos, como texto adicional oculto en la clase .visually-hidden .

Opacity /Opacidad

Agregado en v5.1.0

A partir de la versión 5.1.0, las utilidades de color de texto se generan con Sass mediante variables CSS. Esto permite cambios de color en tiempo real sin compilación y cambios dinámicos de transparencia alfa. .

¿Como funciona?

Consideremos nuestra utilidad predeterminada .text-primary .

    .text-primary {
        --bs-text-opacity: 1;
        color: rgba(var(--bs-primary-rgb), var(--bs-text-opacity)) !important;
    }

Usamos una versión RGB de nuestra variable CSS --bs-primary (con el valor 13, 110, 253 ) y adjuntamos una segunda variable CSS, --bs-text-opacity , para la transparencia alfa (con un valor predeterminado 1 gracias a una variable CSS local). Esto significa que, al usar .text-primary `now`, el valor calculado del color es rgba(13, 110, 253, 1) . La variable CSS local dentro de cada .text-* clase evita problemas de herencia, por lo que las instancias anidadas de las utilidades no modifican automáticamente la transparencia alfa.

Specificity/ Especificidad

A veces, las clases contextuales no se pueden aplicar debido a la especificidad de otro selector. En algunos casos, una solución alternativa suficiente es encapsular el contenido del elemento en uno <div> o más elementos semánticos con la clase deseada.

CSS

Además de la siguiente funcionalidad de Sass, considere leer acerca de nuestras propiedades personalizadas CSS incluidas (también conocidas como variables CSS) para colores y más.

Sass Variables/Variables de Sass

La mayoría de las utilidades de color se generan a partir de nuestros colores temáticos, reasignados desde nuestras variables de paleta de colores genéricas.

    $indigo:  #6610f2;

    $pink:    #d63384;
    $red:     #dc3545;
    $orange:  #fd7e14;
    $yellow:  #ffc107;
    $green:   #198754;
    $teal:    #20c997;
    $cyan:    #0dcaf0;
scss/variables.scss 
    $primary:   $blue;
    $secondary: $gray-600;
    $success:   $green;
    $info:      $cyan;

    $danger:    $red;
    $light:     $gray-100;

Sass maps/Mapas de Sass

Luego, los colores del tema se colocan en un mapa Sass para que podamos recorrerlos para generar nuestras utilidades, modificadores de componentes y más.

scss/_variables.scss 
    $theme-colors: (
        "primary":    $primary,
        "secondary":  $secondary,
        "success":    $success,
        "info":       $info,
        "warning":    $warning,
        "danger":     $danger,
        "light":      $light,
        "dark":       $dark
    );

Los colores en escala de grises también están disponibles como mapa Sass. Este mapa no se utiliza para generar utilidades.

scss/_variables.scss 
    $grays: (
        "100": $gray-100,
        "200": $gray-200,
        "300": $gray-300,
        "400": $gray-400,
        "500": $gray-500,
        "600": $gray-600,
        "700": $gray-700,
        "800": $gray-800,
        "900": $gray-900
        );

Los colores RGB se generan a partir de un mapa Sass independiente:

scss/_maps.scss 
    $theme-colors-rgb: map-loop($theme-colors, to-rgb, "$value");

Las opacidades del color de fondo se basan en esto con su propio mapa que es consumido por la API de utilidades:

scss/_maps.scss 
     $utilities-bg: map-merge(
         $utilities-colors,
        (
            "black": to-rgb( $black),
            "white": to-rgb( $white),
            "body": to-rgb( $body-bg)
        )
    );
     $utilities-bg-colors: map-loop( $utilities-bg,  rgba-css-var, "$key", "bg");
      
     $utilities-bg-subtle: (
        "primary-subtle": var(--#{ $prefix}primary-bg-subtle),
        "secondary-subtle": var(--#{ $prefix}secondary-bg-subtle),
        "success-subtle": var(--#{ $prefix}success-bg-subtle),
        "info-subtle": var(--#{ $prefix}info-bg-subtle),
        "warning-subtle": var(--#{ $prefix}warning-bg-subtle),

        "light-subtle": var(--#{ $prefix}light-bg-subtle),
        "dark-subtle": var(--#{ $prefix}dark-bg-subtle)

Sass utilities API/API de utilidades de Sass

Las utilidades de color se declaran en nuestra API de utilidades en [nombre del archivo scss/_utilities.scss. Aprenda a usar la API de utilidades.

scss/_utilitiess.scss 
            "color": (
            property: color,
            class: text,
            local-vars: (
              "text-opacity": 1
            ),
            values: map-merge(
              $utilities-text-colors,
              (
    "muted":    var(--#{$prefix}secondary-color), // deprecated
    "black-50":    rgba($black, .5), // deprecated
    "white-50":   rgba($white, .5), // deprecated
    "body-secondary":     var(--#{$prefix}secondary-color),
    "body-tertiary": <   var(--#{$prefix}tertiary-color),
    "body-emphasis":  var(--#{$prefix}emphasis-color),
    "reset":    inherit,
              )
            )
          ),
          "text-opacity": (
            css-var: true,
            class: text-opacity,
            values: (
              25: .25,
              50: .5,
              75: .75,
              100: 1
            )
          ),
          "text-color": (
            property: color,
            class: text,
            values: $utilities-text-emphasis-colors
          ),
