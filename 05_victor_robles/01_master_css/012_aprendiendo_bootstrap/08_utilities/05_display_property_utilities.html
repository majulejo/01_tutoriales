05-Display Property-Utilities

DISPLAY/MOSTRAR PROPIEDAD

Cambie rápidamente y con rapidez el valor de visualización de los componentes y mucho más con nuestras utilidades de visualización. Incluye compatibilidad con algunos de los valores más comunes, así como funciones adicionales para controlar la visualización al imprimir.

How it works?/ ¿Como funciona?

Cambie el valor de la propiedad display con nuestras clases de utilidad de visualización adaptables. Admitimos sólo un subconjunto de todos los valores posibles para [nombre del elemento] display. Las clases se pueden combinar para lograr diversos efectos según sus necesidades.

Notation/ Notación

Colorizar el texto con utilidades en color. Si desea colorear enlaces, puede utilizar el .link-*clases de ayudantes que tienen :hover y estados :focus .

Las clases de utilidad de visualización que se aplican a todos los puntos de interrupción , desde xshasta xxl, no tienen abreviatura de punto de interrupción. Esto se debe a que dichas clases se aplican desde min-width: 0;y hacia arriba, y por lo tanto no están vinculadas por una consulta de medios. Sin embargo, los puntos de interrupción restantes sí incluyen una abreviatura de punto de interrupción.

Como tal, las clases se nombran utilizando el formato:

  • .d-{value}paraxs
  • .d-{breakpoint}-{value} para sm, md, lg, xl, y xxl.

Donde "valor" es uno de los siguientes:

  • none
  • inline
  • inline-block
  • block
  • grid
  • inline-grid
  • table
  • table-cell
  • table-row
  • flex
  • inline-flex

Los valores de visualización se pueden modificar modificando los valores display definidos $utilities y volviendo a compilar el SCSS.

Las consultas de medios afectan a anchos de pantalla con el punto de interrupción indicado o mayor . Por ejemplo, con .d-lg-none, se establece display: none , en las pantallas lg,xl y xxl

Hiding elements/ Ocultar elementos

Para un desarrollo más rápido y optimizado para dispositivos móviles, utilice clases de visualización adaptables para mostrar y ocultar elementos según el dispositivo. Evite crear versiones completamente diferentes del mismo sitio; en su lugar, oculte los elementos de forma adaptable para cada tamaño de pantalla.
Para ocultar elementos, simplemente use la clase .d-none o una de las .d-{sm,md,lg,xl,xxl}-none clases para cualquier variación de pantalla responsiva.
Para mostrar un elemento sólo en un intervalo determinado de tamaños de pantalla, puede combinar una clase .d-*-none con una clase .d-*-* ; por ejemplo, .d-none .d-md-block .d-xl-none ocultará el elemento para todos los tamaños de pantalla excepto en dispositivos medianos y grandes.

Display in print/ Mostrar en la impresión

Cambie el valor display de los elementos al imprimir con nuestras clases de utilidad de visualización de impresión. Incluye compatibilidad con los mismos valores display que nuestras utilidades adaptables .d-* .

  • .d-print-none
  • .d-print-inline
  • .d-print-inline-block
  • .d-print-block
  • .d-print-grid
  • .d-print-inline-grid
  • .d-print-table
  • .d-print-table-cell
  • .d-print-table-row
  • .d-print-flex
  • .d-print-inline-flex

Las clases de impresión y exhibición se pueden combinar.

Solo Pantallas (Ocultar solo en impresión)
Solo impresión (Ocultar solo en pantalla)
Ocultar hasta el tamaño máximo en pantalla, pero mostrar siempre en impresión.

HTML

<div class="d-print-none">Solo Pantallas (Ocultar solo en impresión)</div>
<div class="d-none d-print-block">Solo impresión (Ocultar solo en pantalla)</div>
<div class="d-none d-lg-block d-print-block">Ocultar hasta el tamaño máximo en pantalla, pero mostrar siempre en impresión.</div>

Sass utilities API/API de utilidades de Sass

Las utilidades de visualización se declaran en nuestra API de utilidades en [Nombre del archivo] scss/_utilities.scss. Aprenda a usar la API de utilidades. incluidas (también conocidas como variables CSS) para colores y más.

scss/_utilities.scss 
"display": (
    responsive: true,
    print: true,
    property: display 
    class: d,
    values: inline inline-block block grid inline-grid table table-row table-cell flex inline-flex none
  ),
